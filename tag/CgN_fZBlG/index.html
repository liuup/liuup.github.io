NVIDIA Jetson TX2 Notes

2020-01-09 #TX2
NVIDIA Jetson TX2 Notes NVIDIA Jetson TX2 不论在各个方面对于一个新手来说都是一个天坑，而且坑是一个接着一个，我在此记录一些我在配置TX2时遇到的问题，并提出可靠的解决方案。 host： System:Ubuntu 18.04 Lts Processor:Intel® Core™ i5-8300H CPU @ 2.30GHz × 8 Graphics:GeForce GTX 1050 Ti/PCIe/SSE2 Jetson TX2: System:Ubuntu 18.04 Lts Processor:ARMv8 Processor rev 3(v8l) × 4 Graphics:NVIDIA Tegra X2(nvgpu)/integrated 自己主机和TX2配置相差很大，但是刚开始自己并没有意识到这个问题，在更新sources.list时和安装vscode时才意识到这个问题。 TX2的Ubuntu18.04Lts系统是NVIDIA魔改版，用的unity桌面，在界面上会有些不一样。 1.乱七八糟的资料与百科 1.Google镜像站，免翻墙在中国用google的最佳食用方式，Ubuntu改个hosts就行 Google镜像站 2.Jetson TX2 的全英百科，感觉还算全 Jetson TX2 on elinux 3.NVIDIA Jetson TX2 Official NVDIA Jetson TX2 4.TX2接口一览 2.关于刷机 自己的host主机已经安装了NVIDIA SDKManger,将TX2进入恢复模式(同时长按POWER和RECOVERY键两秒后,按一下RESET键,再松开RECOVERY键),在host Terminal上输入lsusb,如果显示出NVIDIA Corp就证明基本连接成功. 2.1 可能遇到的问题 2.1.1 STEP 4 flash时提示没连接上 将进入RECOVERY模式的TX2从host上拔下,直接在host上点刷机(当然,这肯定刷不进去),出来个提示框,第一个模式会让输入TX2的账号和密码,第二个模式是全自动.此时再将TX2连入host,点继续就能成功刷机 2.1.2 刷机过程中提示时间过长,超出预期 点继续就行了 更换数据线,请尽量使用官方数据线,使用质量可靠的数据线能够显著增加数据传输速度 NVIDIA上的关于flash卡99.8%的讨论 值得注意的是,数据线和充电线并不太一样 2.1.3 更换数据线后仍提示检测不到TX2 将TX2彻底关机,将数据线从host上拔出,将电源线拔出,重新连接数据线和电源线,再次进入恢复模式,重新连接数据线和电源线,再次在host上SDKManger中检测 3.关于配置 1.这个地方不注意连sudo apt-get update 都更新不了，为了降低功耗，TX2的CPU架构是arm64(aarch64)，可以通过以下代码查看CPU架构 ~$uname -a 或者 ~$dpkg --print-architecture 2.由于CPU架构的不同，在后面更新sources.list时要做出一些改变 4.Sources 为了加快各个软件的更新下载速度,需要将更新源换成国内 arm ubuntu开发板的apt库和普通ubuntu的库是不一样的，arm用的是ubuntu-ports库。 先获取root权限，再执行 ~$gedit /etc/apt/sources.list 修改source，在此给出Ubuntu 18.04 Lts USTC的源，值得注意的是，arm64的源和普通PC的不一样，要将ubuntu改成ubuntu-ports才能更新 deb http://mirrors.ustc.edu.cn/ubuntu-ports/ bionic-updates main restricted universe multiverse deb-src http://mirrors.ustc.edu.cn/ubuntu-ports/ bionic-updates main restricted universe multiverse deb http://mirrors.ustc.edu.cn/ubuntu-ports/ bionic-security main restricted universe multiverse deb-src http://mirrors.ustc.edu.cn/ubuntu-ports/ bionic-security main restricted universe multiverse deb http://mirrors.ustc.edu.cn/ubuntu-ports/ bionic-backports main restricted universe multiverse deb-src http://mirrors.ustc.edu.cn/ubuntu-ports/ bionic-backports main restricted universe multiverse deb http://mirrors.ustc.edu.cn/ubuntu-ports/ bionic main universe restricted deb-src http://mirrors.ustc.edu.cn/ubuntu-ports/ bionic main universe restricted 更新源 ~$sudo apt-get update 更新之后可能会报GPG Error，我列出Error如下 ~$GPG error: https://packagecloud.io/headmelted/codebuilds/debian stretch InRelease: The following signatures couldn't be verified because the public key is not available: NO_PUBKEY 0CC3FD642696BFC8 解决方案：这里要将公钥改成上面给出的公钥 ~$apt-key adv --recv-keys --keyserver keyserver.ubuntu.com 0CC3FD642696BFC8 如果出现下列结果，说明Error已经消失了 ~$Executing: /tmp/apt-key-gpghome.ZnhRCRjnAd/gpg.1.sh --recv-keys --keyserver keyserver.ubuntu.com 0CC3FD642696BFC8 ～$gpg: key 8D9940AE59B46979: public key &quot;https://packagecloud.io/headmelted/code-oss (https://packagecloud.io/docs#gpg_signing) &lt;support@packagecloud.io&gt;&quot; imported ~$gpg: Total number processed: 1 ~$gpg: imported: 1 然后再更新一下源和软件，这就意味着可以正常从库下载软件了 ~$sudo apt-get update ~$sudo apt-get upgrade 5.VSCODE 紧接着就安装宇宙最强IDE：vscode 我看了好久的Microsoft官方文档，发现官方根本就不支持在linux arm64上面运行，在google上搜索，看到nvidia论坛里也有人问怎么安装vscode。继续以关键词arm64 vscode搜索，发现有人发了如何在树莓派上安装vscode的教程 直接在终端运行，便可以安装vscode ~$sudo apt-get install code-oss 再安装中文包及各种插件，但是会发现c++插件不支持arm64 插件会提示 Architecture arm64 is not supported; &gt;&gt; 来源:C/C++(拓展) 但是能在vscode里用cmake编译，所以个人认为vscode在tx2上并不太好用,推荐安装qt 6.OpenCV 刷机后，TX2自带的是3.3.1，将其刷成3.4.5 刚开始用的16.04Lts arm64源，一直导致好多包安装不了，按照我上面写的换成18.04的源就行 Refenrences： 1.环境配置-Ubuntu 16.04安装opencv3.4.3 - 简书 2.ubuntu下安装opencv3.4.2 - CSDN 3.opencv - github 4.opencv_toncrib - github 可能出现的问题: 提示boostdesc_bgm.i缺少 1.1 编译OpenCV以及openc_contrib提示缺少boostdesc_bgm.i文件出错的解决 - CSDN 1.2 atal error: boostdesc_bgm.i: No such file or directory - Github 提示各种文件路径不对的 直接进入提示错误的文件修改opencv的源代码 在终端使用命令检验是否安装到位 ~$pkg-config --modversion opencv ~$3.4.5 使用g++编译的方法： ~$g++ main.cpp -o main.out `pkg-config opencv --cflags --libs` 7.调用摄像头 7.1调用板载摄像头 终端运行 nvgstcapture-1.0 --cus-prev-res=1920*1080 但是我还没搞懂怎么通过opencv来调用板载摄像头 分辨率可调： 640 * 480 1280 * 720 1920 * 1080 2104 * 1560 2592 * 1944 2616 * 1472 3840 * 2160 3896 * 2192 4208 * 3120 5632 * 3168 5632 * 4224 使用q + 回车退出摄像头 7.2调用usb摄像头 如果想使用usb接口的摄像头，需要将VideoCapture中数值改为1，因为板载摄像头默认是0 VideoCapture capture(1); 8.qt creator 因为vscode在tx2上不太好用，所以需要换个IDE，幸好qt的配置并不难 直接终端运行 ~$sudo apt-get install qt5-default qtcreator -y Reference： NVIDIA
